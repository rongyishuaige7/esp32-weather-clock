# 验证说明

> 状态日期：2026-07-17

## 一键本地门禁

```bash
bash scripts/verify.sh
```

该脚本会将候选导出到临时目录，再执行：

1. 敏感信息、私钥、凭据、私网地址、本机路径、生成物和压缩包检查；
2. 必需文件、PlatformIO 固定版本、BOM、接线 SVG、状态边界与公开范围检查；
3. 不依赖硬件的源码契约测试；
4. ESP32 PlatformIO 干净构建。

它不会修改原工程或历史 ZIP，不会烧录 ESP32、连接真实 Wi-Fi、调用真实天气 API、写入真实凭据或验证实际传感器/显示/网络行为。

## 已知构建探测（仅原始源码）

在公开候选建立前，原始源码的隔离 PlatformIO 探测使用：

```text
PlatformIO Core: 6.1.19
Platform: espressif32 6.13.0
Board target: esp32dev
RAM:   47504 / 327680 bytes (14.5%)
Flash: 1037621 / 1310720 bytes (79.2%)
```

该原始探测不是公开候选的构建证据。2026-07-17 已对当前候选内容运行 `scripts/verify.sh` 并通过：RAM `47488 / 327680`（14.5%）、Flash `1036681 / 1310720`（79.1%）。最终发布仍需以 exact-HEAD GitHub CI 回读为准。约 79% 的 Flash 占用应被视为空间边界，不应忽略。

## 当前真机复测清单

后续复测必须记录日期、完整 Git commit、精确板型、模块/供电事实及每项通过、失败或未测：

- [ ] 确认 ESP32 开发板型号、Flash、USB 芯片、供电和当前提交烧录；
- [ ] 确认 DHT22、BH1750、DS3231、SSD1306 的精确型号、电压、共地、I2C 地址、上拉和物理接线；
- [ ] 无保存 Wi-Fi 时确认开放 AP、`http://192.168.4.1/`、Captive Portal、`/scan`、`/save`、重启与错误输入；
- [ ] 用隔离测试 Wi-Fi 确认 STA、断网、三次重连失败回 AP 和真实 IP（不要公开 IP）；
- [ ] 验证 NTP 成功/失败、RTC 失电默认时间、OLED 初始化/刷新和传感器读数/错误路径；
- [ ] 在受控测试账户下验证天气 API、证书未校验风险、超时、gzip、JSON 解析和显示字段；
- [ ] 连续运行 30–60 分钟，记录重启、内存、Flash 余量、Wi-Fi、RTC、传感器漂移和显示异常；
